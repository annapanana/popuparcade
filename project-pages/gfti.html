Greetings from the Internet

November, 2016
Role: Designer/Developer

Greetings from the Internet (gfti) is a web-based postcard making tool that allows users to search for an image, modify a greetings graphic and include a custom message. The aesthetic of gfti is an homage to Teich greetings postcards. Gfti uses the Flickr API to supply users with images and the Lob API to send postcards.

Process



Ideation and Rough Wireframes

I first came up with the idea for Greetings From the Internet when prompted to brainstorm ideas for a first-quarter project with Galvanize. I had been working as a web developer for roughly 5 weeks and most of that time was spent on front-end development. I wanted to work with images and create an experience that tied web-based development to something physical. A browser-based postcard making tool seemed like an excellent fit for my objectives and skill-level. User Interface

After outlining core features, spending a day expereimenting with my APIs and drafting rough wireframes, I then refined my user interface with digital wireframes. By using a 12-column grid in my digital wireframes, I was able to design for visual consistency across multiple pages and plan for rapid implementation with the materialize css framework. I further refined the interface during development to solve additional UX requirements.


Development Challenges

Here are a few of the challenges that I encountered while developing gfti:

  • Masking images with letters using the p5js library proved to be more difficult than I initially imagined due to the asynchronous retreival of images from Flickr.
  • Saving the postcard composition graphic to propegate across multiple pages with only client-side functionality required me to hack-together a method of saving pixel data in the browser's local storage. Upon loading the image on a new page, the pixel data needed to be redrawn on an HTML canvas object.
  • Before this project, I had never worked with AJAX or API server calls. By spending a day researching my APIs, I felt more confident going into the project. This initial research also helped inform my wireframe/design process.
  • I had also never worked with HTML canvases or the p5js library. The HTML canvases were easy to work with but it took more consideration to configure the asynchronisity of API callbacks (=retrieving images from Flickr) with the draw functionality of p5js.

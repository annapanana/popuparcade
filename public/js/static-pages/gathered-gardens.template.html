Dynamic Vector Graphics: Design Process

Gathered Gardens relied on dynamic SVG maps that I created in Adobe Illustrator

In order to create maps that accurately represent the layout of various gardens, I used Adobe Illustrator to abstract and normalize garden layout diagrams. Because the maps needed to reflect dynamic database information, I exported SVG files of the maps, which could then be visually modified with Angular upon adding or removing produce from a plot. Each plot could include a maximum of 12 different types of produce. The variety of produce in a plot is visually via a dot diagram. Each type of produce had a hex color value associated with it. The colors of the dots in a plot change color depending on what is planted. The diagrams above reflect this process of building up the map visualization.

The same process was applied to different garden layouts:

Plot Data

Users can click on individual plots, represented as colored dot-clusters, to view the produce that is logged in that plot. If the plot belongs to the user, he or she can add and remove produce.

D3.js Data Viz

In addition to visualizing the produce planted in garden maps, we aggregated the data of produce logged to create a bubble chart. The bubble chart visualizes the most common or least commonly planted produce by bubble size and color.

Garden Navigation

The D3 bubble chart visualization included data from 7 different gardens. Users can navigate to different gardens via an interactive, minimalist map menu.

Communication Features

In addition the various data visualization features, users can exchange information using a searchable forum or a real-time chat widget.
